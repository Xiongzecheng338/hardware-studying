GPU 的基本架构

GPU（Graphics Processing Unit，图形处理器）是专门用于图像和图形运算的处理器。与 CPU 不同，GPU 拥有数千个小型计算核心，专为并行处理大量数据而设计。

核心概念：GPU 的并行架构使其在处理图形渲染、科学计算、人工智能等需要大量并行计算的任务时具有显著优势。

1.1 GPU 与 CPU 的架构差异

CPU 架构

  • 少量高性能核心（4-64 核）
  • 强大的单线程性能
  • 复杂的控制逻辑
  • 大容量缓存
  • 适合串行任务

GPU 架构

  • 数千个小型核心
  • 高吞吐量并行计算
  • 简化的控制逻辑
  • 高带宽显存
  • 适合并行任务
特性 CPU GPU
核心数量 4-64 个 1000-10000+ 个
设计目标 低延迟 高吞吐量
缓存大小 大（MB 级） 小（KB 级/核心）
适用场景 通用计算、系统控制 图形渲染、并行计算

1.2 GPU 的核心组件

流处理器（Stream Processors）

GPU 的基本计算单元，负责执行着色器程序。NVIDIA 称为 CUDA 核心，AMD 称为流处理器。每个流处理器都能独立执行计算任务。

显存（VRAM）

专门用于存储图形数据的高速内存，包括纹理、帧缓冲区、几何数据等。现代 GPU 使用 GDDR6/GDDR6X 显存，带宽可达 500GB/s 以上。

张量核心（Tensor Cores）

专门用于矩阵运算的硬件单元，大幅加速深度学习推理和训练。NVIDIA 从 Volta 架构开始引入，RTX 系列标配。

光线追踪核心（RT Cores）

专门用于加速光线追踪计算的硬件单元，可快速计算光线与几何体的交点。NVIDIA RTX 20 系列首次引入。

图形渲染管线

图形渲染管线是 GPU 将 3D 模型转换为 2D 图像的完整流程，包含多个阶段，每个阶段都有特定的处理任务。

1. 应用阶段（Application Stage）

CPU 准备场景数据，包括几何体、材质、光照等信息，并提交给 GPU。

2. 顶点处理（Vertex Processing）

顶点着色器处理每个顶点，执行坐标变换、光照计算等操作。

3. 图元装配（Primitive Assembly）

将顶点组装成三角形、线段等图元，进行裁剪和光栅化准备。

4. 光栅化（Rasterization）

将图元转换为像素片段（fragments），确定哪些像素被覆盖。

5. 像素处理（Fragment Processing）

像素着色器计算每个像素的颜色，应用纹理、光照等效果。

6. 输出合并（Output Merger）

执行深度测试、模板测试、混合等操作，生成最终像素值。

性能瓶颈：图形管线中任何一个阶段都可能成为性能瓶颈。优化时需要分析具体瓶颈所在，而非盲目提升某个阶段的性能。

2.1 着色器类型

顶点着色器

处理每个顶点，执行坐标变换、法线计算、纹理坐标生成等任务。

  • • 模型视图变换
  • • 顶点光照计算
  • • 顶点动画

像素着色器

处理每个像素片段，计算最终颜色值，应用纹理、阴影等效果。

  • • 纹理采样
  • • 光照和阴影
  • • 材质效果

几何着色器

处理图元，可以创建或销毁几何体，实现 tessellation 等功能。

  • • 曲面细分
  • • 几何体生成
  • • 粒子系统

CUDA 与并行计算

CUDA（Compute Unified Device Architecture）是 NVIDIA 推出的通用并行计算架构，允许开发者直接使用 GPU 进行非图形计算任务。

3.1 CUDA 编程模型

// CUDA 内核函数示例
__global__ void vectorAdd(float *a, float *b, float *c, int n) {
    // 计算全局线程 ID
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    
    // 边界检查
    if (i < n) {
        c[i] = a[i] + b[i];
    }
}

// 主机代码调用内核
int main() {
    int n = 1000000;
    int threadsPerBlock = 256;
    int blocksPerGrid = (n + threadsPerBlock - 1) / threadsPerBlock;
    
    vectorAdd<<>>(a, b, c, n);
}

3.2 GPU 并行计算优势

高吞吐量

GPU 可同时执行数千个线程，适合数据并行任务。在矩阵运算、图像处理等场景中，性能可达 CPU 的 10-100 倍。

内存带宽

GDDR6X 显存提供超过 1TB/s 的带宽，远高于 DDR4 内存的 50GB/s，适合大数据量处理。

3.3 通用计算应用场景

深度学习训练

神经网络的前向传播和反向传播都是高度并行的矩阵运算，GPU 可大幅加速训练过程。

科学计算

物理模拟、气象预测、分子动力学等科学计算领域广泛使用 GPU 加速。

视频编解码

GPU 内置的编解码器可硬件加速 H.264、H.265、AV1 等格式的编码和解码。

加密货币挖矿

GPU 的并行计算能力适合某些加密货币的哈希计算，但能耗较高。

光线追踪技术

光线追踪是一种渲染技术，通过模拟光线在场景中的传播路径，生成高度逼真的图像。与传统光栅化相比，光线追踪能更准确地模拟光照、反射、折射等光学现象。

4.1 光线追踪原理

传统光栅化

  • 近似的光照模型
  • 反射效果有限
  • 性能高效
  • 实时渲染可行

光线追踪

  • 物理精确的光照
  • 真实的反射和折射
  • 计算量大
  • RT 核心加速

光线追踪基本算法

for (每个像素) {
    // 从相机发射光线
    Ray ray = createRay(pixel);
    
    // 与场景求交
    HitInfo hit = intersectScene(ray);
    
    if (hit.hit) {
        // 计算光照（包括反射、折射）
        color = calculateLighting(hit);
        
        // 递归追踪反射和折射光线
        if (hit.material.isReflective) {
            Ray reflectedRay = reflect(ray, hit);
            color += traceRay(reflectedRay, depth + 1);
        }
    }
}

4.2 NVIDIA RT 核心

NVIDIA 从 Turing 架构（RTX 20 系列）开始引入 RT 核心，专门用于加速光线追踪计算。RT 核心能快速计算光线与三角形的交点，性能比传统 CUDA 核心高 10 倍以上。

架构 RT 核心版本 光线/秒 代表产品
Turing 第 1 代 5 GRay/s RTX 2060-2080 Ti
Ampere 第 2 代 10 GRay/s RTX 3060-3090
Ada Lovelace 第 3 代 25 GRay/s RTX 4060-4090

性能影响：开启光线追踪会显著降低游戏帧率。建议搭配 DLSS/FSR 等超采样技术使用，以平衡画质和性能。

显存技术详解

显存（Video RAM，VRAM）是 GPU 专用的高速内存，用于存储纹理、帧缓冲区、深度缓冲区、几何数据等图形数据。显存容量和带宽直接影响 GPU 性能。

5.1 显存类型演进

1980s

SRAM / DRAM

早期显卡使用系统内存或专用 SRAM，带宽极低。

1990s

SDRAM

同步动态随机存取内存，带宽提升到 GB/s 级别。

2000s

GDDR3 / GDDR4

图形专用 DDR 内存，带宽和频率大幅提升。

2010s

GDDR5 / GDDR5X

主流显存标准，带宽达到 200-400GB/s。

2018+

GDDR6 / GDDR6X

当前主流，带宽可达 500GB/s - 1TB/s。

2016+

HBM / HBM2

高带宽内存，堆叠式设计，带宽超过 1TB/s。

5.2 显存带宽计算

带宽公式：显存带宽 = 显存频率 × 显存位宽 ÷ 8

例如：GDDR6 显存，频率 14Gbps，位宽 256-bit，带宽 = 14 × 256 ÷ 8 = 448 GB/s

显存类型 典型频率 位宽 带宽 代表显卡
GDDR5 7 Gbps 256-bit 224 GB/s GTX 1070
GDDR6 14 Gbps 256-bit 448 GB/s RTX 3070
GDDR6X 19 Gbps 384-bit 912 GB/s RTX 3090
HBM2 2 Gbps 4096-bit 1024 GB/s RX Vega 64

AI 与深度学习应用

GPU 的并行计算架构使其成为 AI 和深度学习任务的理想选择。现代 GPU 内置的张量核心可大幅加速矩阵运算，这是深度学习的基础。

7.1 张量核心技术

什么是张量核心？

张量核心是专门用于矩阵乘法和累加运算的硬件单元。相比传统 CUDA 核心，张量核心执行 4×4矩阵运算的效率高出 8-12 倍。

应用场景

  • • 深度学习训练和推理
  • • AI 图像生成（Stable Diffusion）
  • • 自然语言处理
  • • 科学计算中的矩阵运算

7.2 DLSS 技术解析

DLSS（Deep Learning Super Sampling）是 NVIDIA 推出的 AI 超采样技术，通过深度学习模型在较低分辨率下渲染游戏，然后 upscale 到高分辨率，大幅提升帧率同时保持画质。

DLSS 版本对比

版本 技术特点 性能提升 代表游戏
DLSS 2.0 时空滤波 2-3 倍 控制、CS:GO
DLSS 3.0 帧生成技术 3-4 倍 赛博朋克 2077
DLSS 3.5 光线重建 4-5 倍 心灵杀手 2

7.3 AI 创作应用

Stable Diffusion

开源 AI 图像生成模型，RTX 3060 12GB 即可流畅运行，生成速度可达 5-10 秒/张。

视频超分辨率

使用 AI 模型将低分辨率视频提升至 4K，Topaz Video AI 等软件依赖 GPU 加速。

语音识别与合成

Whisper、VITS 等 AI 语音模型需要 GPU 进行训练和推理。

显存需求：AI 创作应用对显存容量要求较高。Stable Diffusion XL 建议 12GB 以上显存，大型模型训练需要 24GB+。

选购建议与推荐

选择 GPU 时需要根据预算、用途、显示器分辨率等因素综合考虑。以下是针对不同用户群体的推荐方案。

8.1 按用途选择

游戏玩家

  • 优先关注光栅化性能
  • 光线追踪按需选择
  • 显存 8GB 起步
  • 支持 DLSS/FSR

推荐：RTX 4060 / RX 7700 XT

内容创作者

  • 大显存优先（12GB+）
  • CUDA 生态优势
  • 视频编码能力
  • 稳定性重要

推荐：RTX 4070 Ti / RTX 4080

AI 研究者

  • 张量核心性能
  • 显存容量关键（16GB+）
  • FP16/INT8精度支持
  • NVIDIA 生态优先

推荐：RTX 4080 / RTX 4090

办公用户

  • 集成显卡足够
  • 轻度游戏选入门独显
  • 多显示器支持
  • 低功耗优先

推荐：UHD Graphics / GTX 1650
