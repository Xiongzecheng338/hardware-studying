## 🚀 立即部署到 GitHub

### 方法一：快速部署（推荐新手）

**步骤 1: 创建仓库**
1. 访问 https://github.com/new
2. Repository name: `hardware-studying`
3. 选择 **Public**
4. ❌ 不要勾选 "Initialize this repository with a README"
5. 点击 **Create repository**

**步骤 2: 推送代码**
复制以下命令（替换 YOUR_USERNAME 为你的 GitHub 用户名）：
```bash
git remote add origin https://github.com/YOUR_USERNAME/hardware-studying.git
git branch -M main
git push -u origin main
```

**步骤 3: 启用 GitHub Pages**
1. 进入仓库 Settings → Pages
2. Source: 选择 "Deploy from a branch"
3. Branch: 选择 "main" 和 "/ (root)"
4. 点击 Save

**等待 1-2 分钟，网站就上线了！**

---

### 方法二：使用 GitHub Actions（自动化）

**步骤 1-2: 同上（创建仓库 + 推送代码）**

**步骤 3: 启用 Actions**
1. 进入仓库 Actions 标签
2. 点击 "I understand my workflows, go ahead and enable them"
3. 等待部署工作流完成（2-3 分钟）

**步骤 4: 验证部署**
1. 进入 Settings → Pages
2. 确认 Source 为 "GitHub Actions"
3. 访问显示的 URL

## 🔧 遇到问题？

### 问题 1: 推送代码时出现错误

**错误信息**: `remote: Repository not found.`

**解决方案**:
- 确认仓库名称正确（hardware-studying）
- 确认 GitHub 用户名正确
- 确认仓库已创建且为 Public

---

### 问题 2: 部署后显示 404

**原因**: GitHub Pages 还未完成部署

**解决方案**:
- 等待 2-3 分钟后刷新
- 检查 Actions 标签页查看部署状态
- 确认部署工作流成功完成

---

### 问题 3: 样式/脚本加载失败

**原因**: 文件路径问题

**解决方案**:
- 检查浏览器控制台错误信息
- 确认所有 CSS/JS 文件路径正确
- 使用相对路径（如 `src/css/style.css`）

---

### 问题 4: GitHub Actions 部署失败

**解决方案**:
1. 进入 Actions 标签页
2. 点击失败的部署工作流
3. 查看详细错误信息
4. 根据错误信息修复
